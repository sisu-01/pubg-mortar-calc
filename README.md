# pubg-mortar-calc
배그 박격포 빵야 정확히 높이 딱 크하하

1. FModel 설치 및 디스코드 가입하여 AES Key 확인
2. FModel usmap 했는데도 안돼서 UModel도 실패. 근데 github에 누가 이미지 올려뒀길래 그걸로 three.js 해봄..
3. 121m -> 85도, 700m -> 45도인걸로 ai가 박격포 공식 짜줬음 ㄷㄷㄷ 이제 heightmap이 실제 인게임에서 몇 m로 적용되는지 알아야됨
4. 인게임 스샷 및 공식 검증을 통해 인게임에서 박격포를 0.5 단위로 움직인다는 것을 알았다. 총 80번 움직일 수 있는데 **(최대각도(85)-최소각도(45))/조정회수(80)=0.5** 가 나오고 공식과 비교했을 때 정확하다.<br><br>
   이후 글라이더 타고 배그 지형들 높이 수집하면서(개노가다 ㅠㅠㅠㅠ) heightmap 높이 배율이 96%인 것을 확인. ( Stalber #ffffff가 246m고 밀베가 #7f7f7f 0 등등 이런걸 다양하게 수집했다. )<br>
   예시: ffffff 246,949494 41,8e8e8e 30,8d8d8d 27,28,868686 14,828282 5,808080 1,7f7f7f 0,7e7e7e -1<br><br>
   8x8맵: 8192px -> 8000m, 4x4맵: 4096px -> 4000m<br>
   즉 heightmap scale은 **가로, 세로: 0.9765625**, **높이: 0.96**인 것이다.
5. gdi를 사용하면 전체화면에서도 화면 캡쳐가 된다 ㄷㄷ 하지만 정지 위험이 있다고 한다.<br>
   하지만 DXGI는 없다고 한다. 근데 전체화면 캡쳐가 안되니까 GDI, DXGI 둘 다 설정해둔다.<br>
   openCV 템플릿으로 플레이어 위치와 핑을 검출한다.<br>
   그런데 플레이어와 핑이 지도 격자선과 겹치면 검출이 안되는 문제가 있어서 격자선을 먼저 없애준다.<br>
   지금까지 준비한 것들을 다 합쳐준다.<br>
